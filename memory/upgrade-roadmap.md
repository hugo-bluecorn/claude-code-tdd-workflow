A modernized audit + an evolution review of plugin v2.4.0 produced one prioritized roadmap. Implement items **as TDD slices** (RED→GREEN→REFACTOR), in wave order. Each item has a bashunit "done-test" (the first failing test). Full roadmap + per-item specs: `research/plugin-upgrade/upgrade-roadmap.md` (knowledge-pack workspace) — summarized here.

**Wave 1 — ✅ SHIPPED (2.4.1 → 2.4.5, 2026-05-31).** All seven foundation/correctness items landed (PRs #18–#22, merge-commits): R6 `effort` pinning (2.4.1) · R3 `plugin.json` metadata (2.4.2, unblocked Wave 2) · R2 `Task`→`Agent` denylist + R7 SessionStart `async` + R16 drop inert frontmatter `hooks` (2.4.3) · R15 remove dead `load-role-references.sh` (2.4.4; its old C/C++ `ctest`-in-core slice was DROPPED per [[decisions]] Decision #6 — belongs in the cpp pack) · **R14 genuine 0-failure floor** (2.4.5; settings skip-when-absent + retire/de-brittle ~34 stale doc assertions + `c_documentation_test.sh`→`language_documentation_test.sh`). The suite is now **0 failed local AND CI** ([[test-suite]] [[suite-baseline-corrected]]).

**Wave 2 — distribution & convention re-platform (needs R3 ✅; this is the next frontier):** **R1** (highest leverage on the whole roadmap; **MINOR** — new user-facing pack mechanism) re-platform conventions → `userConfig` + `skills-dir` packs, **data-drive detection** (replace the 4 hardcoded dirnames in `load-conventions.sh`), ship **`dart-flutter-conventions`** as the first pack. **R1 scope EXPANDED per [[decisions]] Decision #6:** (a) **pack-drive the test-command** — turn `hooks/auto-run-tests.sh` (today a hardcoded `.dart→flutter test / .cpp→cmake / .sh→bashunit` chain) into a thin `extension→command` dispatcher reading the active pack(s), with `bashunit` as the built-in default; (b) **remove the now-untested C doc-CONTENT** (the `### C Testing` section in `CLAUDE.md` + the C lines in `tdd-verifier.md`; R14 retired the tests, R1 removes the content) and revisit the Dart/Bash doc-claims. The `dart-flutter` pack = `extensions` + `test-command` (`flutter test {file}`, fvm-aware) + conventions; the C/C++ pack is `claude-cpp-template`'s deliverable, not ours. · R8 publish `.claude-plugin/marketplace.json` · R9 `plugin validate --strict` in CI.

**Wave 3 — quality & job-coverage:** **R4** verifier deterministic re-run **command** gate (replace prompt self-report; today's false-PASS risk) **+ the verifier A/B** ([[decisions]]) → **R4c** codified recallable **test-quality reviewer** (`/tdd-review-tests` + `tdd-test-reviewer`; guards the planner→implementer seam the blackbox verifier can't; reads test+code+spec; rubric [[tdd-review]]; first use = self-audit the plugin's own ~770 tests) → R10 verifier `isolation: worktree` · R5 `/tdd-refactor` (characterize→restructure-under-green; the #1 capability gap) · **R11a retire `/tdd-update-context`** + **R11b rebuild `/tdd-finalize-docs`** (add accuracy-assessment + approval gate) · R12 `/tdd-fix` · R13 `/tdd-green` autopilot.

**Wave 4 — polish & forward bets:** R17 test-first output style · R18 Notification hook · R19 SessionStart TDD reminder · R20 `statusMessage` · **R21** `/tdd-sync-memory` **+ `/tdd-export-memory`** (issue 018; **MINOR**) — cross-session/cross-PROJECT memory transfer. Native CC memory is project-scoped + siloed (no native bridge, no live sync, no official MCP memory server), so replace Hugo's ad-hoc "dump-to-a-dir then RESTART the other session" method with a producer/consumer pair: `/tdd-export-memory` dumps memory-core+context in a standard handoff format; `/tdd-sync-memory` reads it **LIVE (skills re-read disk → no restart needed — the restart only existed because native memory loads at session start)**, proposes promotions into this project's native memory behind an approval gate, then archives the dump. One-way (advisor→plugin), not bidirectional live-sync. ↔ `docs/user-guide.md` promotion model, multi-session-paradigm-survey §8.4/8.5, issue 012 (gate on main thread), R23 (gating becomes per-step) · R22 headless SDK recipe · **R23 per-step autonomy toggle** (issue 017; **MINOR**) — replace the static `disable-model-invocation` gate on the 4 user-gated skills with a default-deny PreToolUse `Skill` hook + a human-only `/tdd-autonomy <steps> on|off|status` setter, so dev/dogfood/headless cycles can run the loop unattended **per step**; relates to issue 012, R13 (`/tdd-green` autopilot), R22; enabler for unattended dogfooding · then evaluate the **deferred** Direction-2 bets (dynamic workflows / agent-teams / channels / routines).
