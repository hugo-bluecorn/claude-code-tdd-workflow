## Overview

### Purpose
Roadmap item R6 (Wave 1, P1, lens C, PRIME=core). Pin a reasoning-`effort` tier in the
YAML frontmatter of every agent under `agents/*.md` so each agent self-declares its cost
tier. None of the 7 agents currently has an `effort` field â€” once pinned, the plugin
self-manages reasoning cost (verifier cheap, planner/implementer deep, the rest balanced).

### Use Cases
- The verifier (blackbox pass/fail) runs at `low` effort â†’ cheap, fast, frequent.
- The planner and implementer run at `high` effort â†’ the two reasoning-heavy seams.
- All other agents default to `medium`.

### Context
This repo IS the tdd-workflow plugin. Agent frontmatter already carries `model`, `maxTurns`,
`tools`/`disallowedTools`, `color`, etc. â€” `effort` is a documented sibling field. Tests in
`test/agents/` parse the frontmatter block with `sed` between the first two `---` markers.

## Requirements Analysis

### Functional Requirements
1. tdd-verifier frontmatter `effort: low`.
2. tdd-planner frontmatter `effort: high`.
3. tdd-implementer frontmatter `effort: high`.
4. Every other present `agents/*.md` declares `effort` in `{low,medium,high}` (= `medium`).
5. The `effort` value lives inside the frontmatter block, not the body.

## Implementation Details

### Architectural Approach
A single trivial slice: add one `effort:` scalar per agent frontmatter. The interesting
design is in the TEST, which must enforce the three explicit tiers while staying decoupled
from the optional role system and from a fixed agent count.

### Design Patterns
- **Explicit-for-core, dynamic-for-rest:** the three core tiered agents are asserted by name
  (they are load-bearing); the remainder are validated by a glob-and-exclude loop so the test
  survives agent add/retire churn and never enumerates role-*.
- **Anti-vacuous guard (Test 6):** assert the dynamic "others" count `>= 1` so a glob/exclusion
  bug can't make Test 4 pass over zero files. Falsifiability over convenience.

### File Structure
```
agents/{tdd-verifier,tdd-planner,tdd-implementer,context-updater,role-creator,tdd-doc-finalizer,tdd-releaser}.md  # +effort
test/agents/agent_effort_test.sh  # NEW
```

### Naming Conventions
Test file: snake_case + `_test.sh`, mirroring `test/agents/`.

## Known Limitations / Trade-offs

### Limitations
- Effort values are static frontmatter; no per-invocation override.

### Trade-offs Made
- Test 4 asserts set-membership `{low,medium,high}` for the "others" rather than strict
  `== medium`. Deliberate: keeps the test from breaking if a future agent legitimately needs a
  different tier, while still enforcing "every agent has a valid effort." The three load-bearing
  tiers are pinned strictly; the rest are validated, not frozen.
