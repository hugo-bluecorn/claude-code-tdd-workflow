2. R7: the SessionStart fetch-conventions hook runs `async: true` (non-blocking session start).

## Implementation Details

### Architectural Approach
Three isolated edits, each guarded by a falsifiable test written first.

- **R2:** single-token swap on one frontmatter line. Test tokenizes `disallowedTools` and
  asserts the standalone `Task` word is gone (not a substring match) and `Agent` present.
- **R7:** add `async: true` to the INNER hook object (`.hooks.SessionStart[fetch].hooks[0]`),
  the same level as `timeout` used elsewhere — per-hook execution modifier. Test asserts
  exactly one `async==true` across the whole file to scope the change.
- **R16:** delete the frontmatter `hooks:` block from two agents; extend the existing
  hooks_json_releaser_test to (a) keep its 6 tests, (b) confirm the canonical Stop gate
  still in hooks.json, (c) assert neither agent frontmatter has `hooks:`, scoping the
  absence check to frontmatter so the doc-finalizer body "hooks" references survive.

### JSON placement decision (R7)
`async: true` → `.hooks.SessionStart[<fetch-conventions group>].hooks[0].async`. Rationale:
every per-hook execution key in hooks.json (e.g. `timeout: 15` on SubagentStop tdd-releaser)
lives on the inner hook object; `async` is the same kind of modifier. The SessionStart group
has no `matcher`; its single inner hook object gains the key.

### File Structure
```

hooks/hooks.json                                       # R7: +async:true on inner hook obj
agents/tdd-releaser.md, agents/tdd-doc-finalizer.md    # R16: remove frontmatter hooks: block
test/agents/context_updater_frontmatter_test.sh        # NEW (R2)

```
